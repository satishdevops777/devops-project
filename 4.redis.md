## üìå What is Redis in RoboShop?

**Redis** is used as an **in-memory data store (cache)** in RoboShop.

It helps by:

* Reducing database load
* Improving application response time
* Storing frequently accessed data in memory

In RoboShop, backend services (like **User**, **Cart**, etc.) access Redis over the network.

## 1Ô∏è‚É£ Install Redis (Using DNF Modules)

### Why DNF Modules?

RHEL-based systems support **multiple Redis versions** using module streams. We explicitly enable Redis **6.x** to match application compatibility.

---

### Disable default Redis module

```bash
dnf module disable redis -y
```

This avoids installing an older or unsupported version.

---

### Enable Redis 6.x module

```bash
dnf module enable redis:6 -y
```

This ensures Redis 6.x (6.2 compatible) is installed.

---

### Install Redis

```bash
dnf install redis -y
```

Verify installation:

```bash
redis-server --version
```

---

## 2Ô∏è‚É£ Why Redis Is Not Accessible by Default

By default, Redis listens only on:

```text
127.0.0.1 (localhost)
```

### What this means

* Only applications running **on the same server** can connect
* Remote application servers **cannot access Redis**

This is a **secure default**, but not suitable for RoboShop.

---

## 3Ô∏è‚É£ Update Redis Listen Address

### Why change to `0.0.0.0`?

`0.0.0.0` means:

> ‚ÄúAccept connections from **any network interface**‚Äù

This allows other RoboShop services to connect to Redis.

---

### Files to update

Redis configuration may exist in **two locations** (depends on OS/package):

* `/etc/redis.conf`
* `/etc/redis/redis.conf`

---

### Edit configuration files

```bash
vim /etc/redis.conf
vim /etc/redis/redis.conf
```

### Change this line

```ini
bind 127.0.0.1
```

### To

```ini
bind 0.0.0.0
```

> ‚ö†Ô∏è Ensure both config files are updated if they exist.

## üîê Security Note (Very Important)

Exposing Redis publicly **without protection is dangerous**.

In real production environments, Redis should be:

* Accessible only inside a **private network (VPC)**
* Protected by **firewall / security groups**
* Optionally secured using **authentication (`requirepass`)**

RoboShop labs keep it simple for learning purposes.
